# Advanced Plan Execution

Master DojOps plan execution with dry runs, resume, retry, replay, rollback, and targeted task execution.

---

## Overview

In this tutorial, you'll learn the advanced `dojops apply` options:

- Preview changes with `--dry-run` before writing files
- Resume partially-failed plans with `--resume` and `--retry`
- Run deterministic replays with `--replay`
- Execute individual tasks with `--task`
- Manage timeouts, force mode, and path overrides
- Roll back and clean up generated artifacts
- Inspect execution history and audit trails

**Time estimate:** ~15 minutes

**Prerequisites:**

- DojOps installed and configured
- A project initialized (`dojops init`)
- A saved plan (run `dojops plan "your goal"` first)

## Step 3: Resume a Failed Plan

If a plan partially fails (e.g., network timeout on task 2), resume from where it left off:

```bash
dojops apply --resume
```

```
Ôöî  Resuming Plan: plan-f7a2b1c3
Ôöé
Ôöé  Task 1/3: github-actions ÔÇö Ôťô already completed
Ôöé  Task 2/3: dockerfile ÔÇö retrying...
Ôöé  Generated: Dockerfile
Ôöé  Ôťô Written: Dockerfile
Ôöé
Ôöé  Task 3/3: docker-compose
Ôöé  Generated: docker-compose.yml
Ôöé  Ôťô Written: docker-compose.yml
Ôöé
Ôöö  Plan complete ÔÇö 3/3 tasks succeeded (1 resumed)
```

Resume skips already-completed tasks. Combine with `--yes` for unattended recovery:

```bash
dojops apply --resume --yes
```

### Retry Failed Tasks

By default, `--resume` skips failed tasks and continues with unblocked ones. Add `--retry` to re-attempt failed tasks:

```bash
dojops apply --resume --retry
```

```

```


## Step 6: Timeouts and Force Mode

### Per-Task Timeout

Set a custom timeout for LLM generation (default: 60 seconds):

```bash
dojops apply --timeout 120
```

### Force Mode

Skip the git dirty working tree check, HIGH risk confirmation gate, and replay validation:

```bash
dojops apply --force
```

This is useful in CI environments where the working directory may have uncommitted changes. Use with caution.

### Skip Verification

Skip external tool verification (e.g., `terraform validate`, `actionlint`):

```bash
dojops apply --skip-verify
```

### Allow All Paths

Bypass the DevOps file write allowlist (for non-standard output locations):

```bash
dojops apply --allow-all-paths
```

### Install Packages After Apply

Automatically run the package manager install after a successful apply:

```bash
dojops apply --install-packages
```

## Quick Reference

| Flag                 | Description                                              |
| -------------------- | -------------------------------------------------------- |
| `--dry-run`          | Preview changes without writing files                    |
| `--yes`              | Auto-approve all confirmations                           |
| `--resume`           | Resume from last completed task                          |
| `--retry`            | Retry failed tasks (use with `--resume`)                 |
| `--replay`           | Deterministic: temperature=0, validate environment match |
| `--force`            | Skip git dirty check, risk gate, replay validation       |
| `--task <id>`        | Execute only one task from the plan                      |
| `--timeout <sec>`    | Per-task timeout in seconds (default: 60)                |
| `--skip-verify`      | Skip external config verification                        |
| `--allow-all-paths`  | Bypass DevOps file write allowlist                       |
| `--install-packages` | Run package manager install after apply                  |
