# Custom Agents & Skills

Create custom specialist agents, build custom skills, and publish to the DojOps Hub.

---

## Overview

In this tutorial, you'll learn to:

- Create custom specialist agents (LLM-generated and manual)
- Browse, inspect, and remove agents
- Scaffold, validate, and test custom skills
- Search, install, and publish skills on the DojOps Hub

**Time estimate:** ~20 minutes

**Prerequisites:**

- DojOps installed and configured with an LLM provider
- A project initialized (`dojops init`)
- For publishing: a DojOps Hub account and API token

---

### Step 3: Create a Custom Agent (LLM-Generated)

Create a new specialist agent using AI:

```bash
dojops agents create "an SRE specialist for incident response and reliability engineering"
```

```
┌  Creating Custom Agent
│
│  Generating agent configuration...
│
│  Name: sre-specialist
│  Domain: reliability
│  Description: Expert in incident response, SLOs, error budgets,
│               chaos engineering, and production reliability
│
│  Keywords: sre, incident, postmortem, slo, sla, error-budget,
│            reliability, chaos, runbook, on-call, pagerduty,
│            observability, toil, capacity
│
│  Saved to: .dojops/agents/sre-specialist/README.md
│
└  Agent created — available immediately
```

The agent is now active and participates in keyword routing:

```bash
dojops "Create a runbook for database failover incidents"
# Routes to sre-specialist automatically
```

## Part 2: Custom Skills

### Step 6: Scaffold a New Skill

Create a new custom skill as a `.dops` file:

```bash
dojops skills init my-nginx-proxy
```

```
┌  Skill Scaffolded
│
│  Created: .dojops/skills/my-nginx-proxy.dops
│
│  Edit the .dops file to customize your skill.
│
└  Ready to customize
```

The generated `.dops` file is a single self-contained file combining YAML frontmatter (metadata, input/output schemas, file specs, permissions) with markdown prompt sections. Open it in your editor to customize the prompt, input fields, and output format.

### Step 7: List Discovered Skills

View all custom skills (global + project-scoped):

```bash
dojops skills list
```

```
┌  Custom Skills
│
│  Project (.dojops/skills/):
│  ├  my-nginx-proxy  v0.1.0  — (scaffold template)
│
│  Global (~/.dojops/skills/):
│  (none)
│
└  1 project, 0 global
```

### Step 8: Validate a Skill

Check that your skill is valid:

```bash
dojops skills validate my-nginx-proxy
```

```
┌  Skill Validation
│
│  ✓ DOPS skill is valid: my-nginx-proxy v0.1.0
│  Format: .dops
│  Files: 1 output file defined
│
└  Skill is valid
```

### Step 9: Load a Skill from Disk

Import a `.dops` skill or legacy tool directory from any local path into your project:

```bash
dojops skills load /path/to/shared-tools/redis-config/
```

```
┌  Skill Loaded
│
│  Copied to: .dojops/tools/redis-config/
│
└  Skill available for use
```

## Part 3: DojOps Hub

### Step 10: Search for Skills

Search the [DojOps Hub](https://hub.dojops.ai) for community skills:

```bash
dojops skills search docker
```

```
┌  Hub Search: "docker"
│
│  docker-swarm        v1.2.0  ★ 45  ↓ 230  Docker Swarm stack configs
│  docker-rootless     v1.0.0  ★ 23  ↓ 112  Rootless Docker daemon setup
│  docker-registry     v2.1.0  ★ 18  ↓ 89   Private Docker registry config
│
└  3 results
```

Refine your search:

```bash
dojops skills search terraform --limit 5
dojops skills search k8s --output json
```

### Step 11: Install a Skill from Hub

Install a community skill for immediate use:

```bash
dojops skills install docker-swarm
```

```
┌  Skill Installed
│
│  Package: docker-swarm v1.2.0
│  Integrity: SHA-256 verified ✓
│  Location: .dojops/tools/docker-swarm.dops
│
└  Skill ready to use
```

Install a specific version globally (shared across all projects):

```bash
dojops skills install docker-swarm --version 1.0.0 --global
```

### Step 12: Publish a Skill to Hub

First, set up your Hub API token:

```bash
# 1. Sign in at hub.dojops.ai → Settings → API Tokens
# 2. Generate a token (format: dojops_<40-hex-chars>)
# 3. Set the environment variable:
export DOJOPS_HUB_TOKEN="dojops_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2"
```

Publish your `.dops` skill:

```bash
dojops skills publish my-nginx-proxy.dops
```

```
┌  Publishing to DojOps Hub
│
│  Package: my-nginx-proxy v1.0.0
│  SHA-256: a1b2c3d4...
│  Size: 2.4 KB
│
│  ✓ Uploaded successfully
│  URL: https://hub.dojops.ai/packages/my-nginx-proxy
│
└  Published
```

Include a changelog with your release:

```bash
dojops skills publish my-nginx-proxy.dops --changelog "v1.1.0: Added SSL termination support"
```
