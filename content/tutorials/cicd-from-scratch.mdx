# CI/CD from Scratch

Build a full CI/CD pipeline from zero using DojOps plan and apply.

---

## Overview

In this tutorial, you'll start with a bare Node.js project that has zero DevOps files and use DojOps to:

- Plan a complete CI/CD pipeline using task graph decomposition
- Review and validate the plan before execution
- Execute the plan with approval workflows
- Verify the results with a quality check and audit trail

**Time estimate:** ~20 minutes

**Prerequisites:**

- Node.js >= 20 installed
- An API key from a supported LLM provider
- Git installed

---

## Step 1: Setup

Install DojOps and create a minimal project:

```bash
npm i -g @dojops/cli
```

Create a fresh Node.js project to work with:

```bash
mkdir my-app && cd my-app
git init
npm init -y
```

Add a simple build script to `package.json`:

```json
{
  "scripts": {
    "build": "echo 'Building...'",
    "test": "echo 'Tests passed'",
    "start": "node index.js"
  }
}
```

Create a minimal `index.js`:

```bash
echo 'console.log("Hello from my-app")' > index.js
```

Configure your LLM provider if you haven't already:

```bash
dojops provider add openai --token sk-...
```

---

## Step 2: Initialize

Run `dojops init` to scan the project:

```bash
dojops init
```

```

  (none detected)

```

DojOps correctly identifies this as a bare project with no CI/CD or container infrastructure.

---

## Step 3: Plan the CI/CD

Use `dojops plan` to decompose a high-level goal into a dependency-aware task graph:

```bash
dojops plan "Set up CI/CD for a Node.js app with GitHub Actions, Docker, and Docker Compose"
```

DojOps sends your goal to the LLM along with the project context, and receives a structured task graph:

```
Planning: "Set up CI/CD for a Node.js app with GitHub Actions, Docker, and Docker Compose"

Task Graph (3 tasks):
  1. [github-actions] Create GitHub Actions CI workflow
     Dependencies: none
  2. [dockerfile] Create multi-stage Dockerfile
     Dependencies: none
  3. [docker-compose] Create Docker Compose for local development
     Dependencies: [2] dockerfile

Execution order (topological):
  1. github-actions + dockerfile (parallel)
  2. docker-compose (after dockerfile)

Plan saved to .dojops/plans/plan-<id>.json
```

The planner decomposes your goal into individual tool-scoped tasks, identifies dependencies (Docker Compose depends on the Dockerfile), and determines the optimal execution order.

---

## Step 4: Review the Plan

Before executing, review what DojOps will generate:

```bash
dojops validate
```

```
Validating plan plan-<id>...

  Task 1: github-actions
    Tool:    github-actions-tool
    Action:  create .github/workflows/ci.yml
    Status:  valid

  Task 2: dockerfile
    Tool:    dockerfile-tool
    Action:  create Dockerfile
    Status:  valid

  Task 3: docker-compose
    Tool:    docker-compose-tool
    Action:  create docker-compose.yml
    Depends: task 2
    Status:  valid

All tasks validated successfully.
```

You can also inspect the last plan in detail:

```bash
dojops explain last
```

This shows the full task graph with inputs, expected outputs, and the reasoning behind the decomposition.

---

## Step 5: Execute

Apply the plan with the interactive approval workflow:

```bash
dojops apply
```

DojOps executes each task in topological order, asking for approval at each step:

```
Executing plan plan-<id>...

[1/3] github-actions: Create GitHub Actions CI workflow

  Generated: .github/workflows/ci.yml
    - Node.js 20/22 matrix
    - Install, build, test, lint steps
    - Dependency caching
    - Security audit

  Apply this file? (y/n/diff): y
  Written: .github/workflows/ci.yml

[2/3] dockerfile: Create multi-stage Dockerfile

  Generated: Dockerfile
    - Multi-stage build (deps → builder → runner)
    - Node 20-slim base
    - Non-root user
    - Optimized layer caching

  Apply this file? (y/n/diff): y
  Written: Dockerfile

[3/3] docker-compose: Create Docker Compose config

  Generated: docker-compose.yml
    - App service (builds from Dockerfile)
    - Port mapping (3000:3000)
    - Volume mount for development
    - Environment variables

  Apply this file? (y/n/diff): y
  Written: docker-compose.yml

Plan executed successfully. 3 files created.
```

Choosing `diff` at any prompt shows the exact content that will be written, so you can review before committing.

For non-interactive environments, use `--yes` to auto-approve all operations:

```bash
dojops apply --yes
```

---

## Step 6: Verify Results

Run the quality check again to see the improvement:

```bash
dojops check
```

```
DevOps Quality Check

Score: 78/100 (Good)

Findings:
  info  Consider adding a SECURITY.md
  info  Consider adding Dependabot or Renovate for dependency updates
  info  Consider adding a Makefile for common tasks

Missing files:
  SECURITY.md
  CODEOWNERS
  renovate.json or dependabot.yml
```

The score jumped from baseline to 78 — the critical CI/CD and container infrastructure is now in place.

Check the audit trail to see what happened:

```bash
dojops history list
```

```
ID        Type       Tool               Status     Date
plan-001  plan       planner            completed  2025-01-15 10:30
exec-001  execute    github-actions     completed  2025-01-15 10:31
exec-002  execute    dockerfile         completed  2025-01-15 10:31
exec-003  execute    docker-compose     completed  2025-01-15 10:32
```

Each execution is recorded with a hash-chained audit entry. Verify the chain integrity:

```bash
dojops history verify
```

```
Audit chain integrity: verified
Entries: 4
Chain status: valid (all hashes match)
```
